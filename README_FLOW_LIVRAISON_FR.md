# Flow livraison, problemes et evaluations

## 1. Acheteur

- L'acheteur consulte son dashboard avec des cartes accessibles.
- Il peut rechercher et filtrer ses commandes.
- Dans le suivi de livraison, chaque commande affiche une carte avec :
  - les informations globales de la commande ;
  - les produits ;
  - l'agriculteur ;
  - l'adresse de livraison ;
  - les informations du transporteur ;
  - un bouton pour contacter le livreur ;
  - un bouton pour marquer la commande comme livree ;
  - un bouton pour signaler un probleme.

## 2. Transporteur

- Quand le transporteur marque une commande comme livree, l'acheteur recoit une notification.
- L'acheteur peut ensuite confirmer la reception, evaluer la commande ou signaler un probleme.

## 3. Evaluation apres livraison

Quand la commande est livree, l'acheteur peut laisser une evaluation pour :

- l'agriculteur ;
- le transporteur ;
- le produit.

Apres l'envoi de l'evaluation, l'agriculteur recoit une notification dans son dashboard.

## 4. Signalement d'un probleme

Si l'acheteur clique sur Signaler un probleme :

- il ecrit un message qui explique le probleme ;
- le signalement est enregistre ;
- l'agriculteur recoit une notification avec le message de l'acheteur.

## 5. Agriculteur

Le dashboard agriculteur affiche les alertes acheteurs :

- problemes signales ;
- nouvelles evaluations ;
- details de la commande concernee ;
- message de l'acheteur ;
- notes recues.
